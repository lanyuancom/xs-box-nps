# FAQ

- 服务端无法启动
```
服务端默认配置启用了8024，8080，80，443端口，端口冲突无法启动，请修改配置
```
- 客户端无法连接服务端
```
请检查配置文件中的所有端口是否在安全组，防火墙放行
请检查vkey是否对应
请检查版本是否对应
```
- 服务端配置文件修改无效
```
install 之后，Linux 配置文件在 /etc/nps
```
- p2p穿透失败 [p2p服务](https://ehang-io.github.io/nps/#/example?id=p2p%e6%9c%8d%e5%8a%a1)
